# Iron Hand Services - Landing Page

A professional, responsive landing page for a handyman services business. Built as a portfolio project demonstrating modern web development skills.

## 🎯 Project Overview

This is a multi-page website showcasing a fictional handyman service company. The site features clean design, smooth animations, and full mobile responsiveness.

## 🚀 Features

- **Responsive Design** - Works seamlessly on desktop, tablet, and mobile
- **Multi-Page Navigation** - 5 interconnected pages with consistent styling
- **Interactive Elements** - Hover effects, smooth scrolling, and page transitions
- **Form Validation** - Working contact form with success/error messaging
- **Accessibility** - Keyboard navigation, focus states, and ARIA labels
- **Modern CSS** - Grid, Flexbox, animations, and custom properties
- **Clean JavaScript** - ES6+ features, event handling, and DOM manipulation

## 📂 Project Structure
```
iron-hand-services/
├── index.html          # Homepage with hero, services preview, testimonials
├── about.html          # Company story, values, service areas, certifications
├── services.html       # Detailed service offerings
├── gallery.html        # Project photo gallery with overlays
├── contact.html        # Contact form and business information
├── ironHand.css        # Main stylesheet (shared across all pages)
├── ironHand.js         # Main JavaScript (navigation, scroll effects, forms)
├── services.css        # Services page specific styles
├── gallery.css         # Gallery page specific styles
├── contact.css         # Contact page specific styles
├── about.css           # About page specific styles
├── favicon.ico         # Browser tab icon
├── Tool Table.jpg      # Hero section background image
└── photo1-10.jpg       # Gallery images
```

## 📱 Responsive Breakpoints

- **Desktop**: 1200px+ (full layout)
- **Tablet**: 768px - 1199px (adjusted grid)
- **Mobile**: < 768px (hamburger menu, stacked layout)

## 🔧 Setup Instructions

1. Download all project files to a local folder
2. Ensure all images are in the same directory as HTML files
3. Open `index.html` in a web browser
4. No build process or dependencies required!
